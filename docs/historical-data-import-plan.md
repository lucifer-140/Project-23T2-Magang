# Historical Data Import Plan

## Overview

Four-phase pipeline to seed legacy academic documents into the system.

```
Phase 1 (done)   → Reorganize files/ into canonical layout ✓
Phase 2 (script) → Rename dosen folders to canonical names
Phase 3 (you)    → Fill import-data/metadata/ CSVs
Phase 4 (script) → Import everything into DB
```

---

## Canonical Folder Layout

```
files/
└── {YYYY_YYYY}/                          e.g. 2024_2025
    └── {Genap|Ganjil|Akselerasi}/
        └── {DosenName}/                  canonical name, no academic titles
            └── {ClassName}_{MatkulName}/ e.g. 22TI1_Inovasi Teknologi dan Bisnis
                ├── RPS.pdf
                ├── LPP.pdf
                ├── SOAL_UTS.pdf
                ├── SOAL_UAS.pdf
                ├── EPP_UTS.pdf
                └── EPP_UAS.pdf
```

## Metadata CSVs

Stored in `import-data/metadata/`. Generated by `scripts/generate-csv-templates.ts`, filled by you.

### `users.csv`

```csv
name,email,password,roles
Ade Maulana,ade.maulana@uph.edu,dosen123,DOSEN
Robin,robin@uph.edu,dosen123,DOSEN|KOORDINATOR
```

- `name` = exact canonical folder name (case-sensitive)
- `roles` = pipe-separated values: `DOSEN`, `DOSEN|KOORDINATOR`, `DOSEN|KAPRODI`, `DOSEN|PRODI`
- Script upserts by email — safe to re-run

### `matkul.csv`

```csv
name,code,sks
Interaksi Manusia & Komputer,IMK301,3
Algoritma & Pemograman,ALP101,3
```

- `name` = exact match of matkul portion in folder names
- `code` = unique short code
- `sks` = integer credit hours

---

## What the Import Script Does

```
1. Read users.csv     → upsert User (status=ACTIVE)
2. Read matkul.csv    → upsert KatalogMatkul
3. Walk files/{year}/ → upsert TahunAkademik (isActive=false for historical years)
4. Walk {sem}/        → upsert Semester
5. Walk {DosenName}/  → resolve User by name (hard error if missing from CSV)
6. Walk {Class}_{Matkul}/
     → upsert Matkul (linked to semester + katalog)
     → upsert MatkulClass
     → connect dosen to Matkul + MatkulClass
7. For each doc file (non-RPS):
     → copy to public/uploads/historical/
     → upsert AcademicDocument (status=APPROVED, isKoordinatorApproved=true, isProdiApproved=true)
8. For RPS.pdf:
     → copy to public/uploads/historical/
     → upsert RPS (status=APPROVED, isKoordinatorApproved=true)
```

`--dry-run` prints all planned operations without touching DB or filesystem.
